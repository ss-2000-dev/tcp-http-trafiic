# TCP-HTTP 通信の理解を深めるためのドキュメント

## データ形式関連

### エンコード

- 人間が読める文字（文字列）からコンピュータが扱える数値（バイト列）に変換すること
- ex.「A」→「01000001」
- UTF-8：変換のルール（文字コード）

### Base64

- 「バイナリデータ（0〜255 の数値列）を、A〜Z, a〜z, 0〜9, +, / の 64 種類の文字で表す」エンコード方式

  - 任意のバイナリデータを通信で安全に扱える ASCII 文字 に変換する方法

- 「バイナリデータを安全に文字列化する」ための方式 → 入力として受け取るのは「バイト列（bytes）」
- なぜ必要か？：TCP や HTTP などのプロトコルは「バイト列」を扱いますが、特に HTTP（テキストベース）では、制御文字や改行などがデータに混ざると壊れることがある
  - そのため、画像や音声、バイナリなどを送るときに「安全な ASCII テキストに変換して送ろう！」という目的で Base64 が使われる
- バイト列、バイナリデータ

#### 文字コード

- 「どの文字をどんな数値に対応させるか」を決める**ルール（変換表）**

#### バイト列

- 基本 16 進数で表される
